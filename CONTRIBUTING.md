# Contributing

Thank you for helping improve this repository.

## Ways to contribute
- Propose new workflow templates
- Improve docs and examples
- Share connector mappings and implementation notes
- Report issues and suggest governance improvements

## Contribution process
1. Fork or branch from `main`.
2. Make focused, well-scoped changes.
3. Update docs and examples when behavior or structure changes.
4. Open a PR with clear context, before/after, and impact.

## Quality checklist
- Template has Goal, Inputs, Workflow, Outputs, KPIs.
- Documentation is concise and copy-paste friendly.
- File names use lowercase kebab-case.
- Content is vendor-neutral unless explicitly noted.
