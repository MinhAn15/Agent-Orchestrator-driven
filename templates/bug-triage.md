# Bug Triage Workflow Template

## Goal
Prioritize defects quickly and route issues to the right owners with clear severity.

## Inputs
- Incoming bug reports (ticket, issue tracker, logs)
- Reproduction steps and environment data
- Product component ownership map

## Workflow (Copy/Paste)
1. Validate report completeness (steps, expected vs actual, logs).
2. Attempt reproduction and record reproducibility level.
3. Classify severity (S1-S4) and impact scope.
4. Map issue to owning team/component.
5. Suggest immediate mitigation or workaround.
6. Create/Update engineering ticket with priority and evidence.

## Output
- Triage summary
- Priority + owner assignment
- Linked issue with reproduction artifacts

## KPIs
- Time to triage
- Reopen rate
- SLA compliance by severity
