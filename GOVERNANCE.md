# Governance

## Maintainer responsibilities
- Curate high-quality, reusable templates.
- Review contributions for clarity, safety, and neutrality.
- Keep documentation current and easy to adopt.

## Decision-making
- Minor edits: maintainer consensus in PR review.
- Major changes (new domains, structure shifts): documented proposal + review window.

## Release strategy
1. **Weekly changelog**
   - Publish a concise changelog every week summarizing merged improvements, fixes, and docs updates.
2. **Monthly showcase**
   - Highlight successful implementations, notable contributors, and top template enhancements.
3. **Community template of the month**
   - Select one community-submitted template monthly based on impact, clarity, and reusability.
   - Feature the winner in docs and encourage iteration from community feedback.

## Community participation
Contributors are encouraged to suggest new release rituals and voting mechanisms as adoption grows.
